# トータルリサーチアプリ 要件定義書

## 1. 概要

キーワードを入力し、リサーチパターンを選択することで、AI（Claude）が構造化されたリサーチデータをスプレッドシート形式で生成するWebアプリケーション。

---

## 2. 機能要件

### 2-1. キーワード入力

| 項目 | 仕様 |
|------|------|
| 入力方式 | テキストボックス（1行） |
| 文字制限 | 1〜100文字 |
| バリデーション | 空白のみ不可、空文字の場合は実行ボタンを無効化 |
| 例 | 「ダイエット」「AI活用」「マーケティング自動化」 |

### 2-2. リサーチパターン（プルダウン選択）

| ID | パターン名 | 説明 | 出力列数 |
|----|-----------|------|---------|
| seo | SEOキーワードリサーチ | 関連キーワード・検索意図・コンテンツ案 | 6列 |
| competitor | 競合リサーチ | 強み・弱み・差別化ポイント・対抗戦略 | 6列 |
| content | コンテンツ企画 | タイトル・見出し構成・ターゲット | 6列 |
| market | 市場・業界リサーチ | 市場規模・トレンド・機会・リスク | 6列 |
| sns | SNS・バズリサーチ | ハッシュタグ・投稿アイデア・プラットフォーム別戦略 | 6列 |
| persona | ペルソナ分析 | ターゲット顧客・課題・購買心理・情報収集行動 | 7列 |

#### 各パターンの出力列定義

**SEOキーワードリサーチ**
- メインキーワード / 関連キーワード / 検索意図 / 月間検索ボリューム(推定) / 競合難易度(1-10) / コンテンツ案

**競合リサーチ**
- 競合名 / カテゴリ / 強み / 弱み / 差別化ポイント / 対抗戦略

**コンテンツ企画**
- タイトル案 / H2見出し / H3見出し / ターゲットキーワード / 想定読者 / 推定文字数

**市場・業界リサーチ**
- 分析軸 / 詳細 / 数値・規模 / トレンド方向 / ビジネス機会 / リスク

**SNS・バズリサーチ**
- プラットフォーム / ハッシュタグ / バズコンテンツ傾向 / 投稿アイデア / 最適投稿時間帯 / 期待エンゲージメント

**ペルソナ分析**
- ペルソナ名 / 年齢層・性別 / 職業・役職 / 課題・悩み / ニーズ・欲求 / 購買動機 / 情報収集チャネル

### 2-3. 出力形式（選択制）

| 形式 | 動作 | 備考 |
|------|------|------|
| テーブル表示 | ブラウザ内にスプレッドシート形式でリアルタイム表示 | デフォルト |
| CSVダウンロード | UTF-8 BOM付きCSVファイルとして保存 | Excel対応 |
| Excelダウンロード | .xlsxファイルとして保存、列幅自動調整 | |
| JSONダウンロード | 整形済みJSONファイルとして保存 | API連携用途 |

- 実行後もテーブル表示画面からCSV/Excel/JSONへの追加エクスポートが可能
- ファイル名: `{キーワード}_{パターン名}_{YYYY-MM-DD}.{拡張子}`

### 2-4. リアルタイムストリーミング

- AIの応答をストリーミングで受信し、完成したレコードから順次テーブルに表示する
- ローディング中はアニメーション表示とストリームテキストのプレビューを表示
- ストリーミング中でも部分的なテーブルデータを表示する（パーシャルJSON解析）

## 3. 非機能要件

### 3-1. AI生成データ量

| パターン | 最低生成行数 |
|---------|------------|
| SEOキーワードリサーチ | 20行以上 |
| 競合リサーチ | 15行以上 |
| コンテンツ企画 | 15記事以上 |
| 市場・業界リサーチ | 20行以上 |
| SNS・バズリサーチ | 20行以上 |
| ペルソナ分析 | 15人以上 |

### 3-2. 使用モデル・API

- AIモデル: `claude-sonnet-4-6`
- 最大トークン: 8,000（出力）
- ストリーミング: 有効
- APIキー: サーバーサイドのみで使用（フロントエンドに露出しない）

### 3-3. エラーハンドリング

| エラー種別 | 表示方法 |
|-----------|---------|
| APIエラー | 赤色バナーでメッセージ表示 |
| JSON解析エラー | フォールバック（空配列）、エラー表示なし |
| ネットワークエラー | 赤色バナーでメッセージ表示 |
| 入力値不正 | ボタン無効化 |

### 3-4. UX

- モバイル・デスクトップ対応（レスポンシブ）
- 横スクロール対応テーブル（列数が多い場合）
- 行ホバー時のハイライト
- ローディング中はフォームを無効化（二重送信防止）

## 5. ファイル構成

```
/
├── app/
│   ├── page.tsx                  # トップページ
│   ├── layout.tsx                # レイアウト
│   ├── globals.css               # グローバルスタイル
│   └── api/research/route.ts     # リサーチAPIエンドポイント
├── components/
│   ├── ResearchApp.tsx           # メインアプリ（状態管理）
│   ├── ResearchForm.tsx          # 入力フォーム
│   └── ResultsTable.tsx          # 結果テーブル
├── lib/
│   ├── research-patterns.ts      # パターン定義・プロンプト
│   └── export-utils.ts           # エクスポート処理
└── .env.local                    # 環境変数（APIキー）
```

---

## 6. 環境変数

| 変数名 | 必須 | 説明 |
|--------|------|------|
| ANTHROPIC_API_KEY | 必須 | Anthropic APIキー |

`.env.local` に設定:
```
ANTHROPIC_API_KEY=sk-ant-...
```

---

## 7. セットアップ手順

```bash
# 依存関係インストール
npm install

# 環境変数設定
# .env.local の ANTHROPIC_API_KEY を実際のAPIキーに書き換える

# 開発サーバー起動
npm run dev
# → http://localhost:3000 でアクセス
```
